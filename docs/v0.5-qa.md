# v0.5 — Word Card — QA Checklist

Overview
Use this checklist to manually verify the v0.5 Word Card feature.

Manual QA
1. Open `/#/explore` — vocabulary list renders.
2. Click a word card — card expands in-place and shows fields (hu, pl, en, type, pattern, family, related, tags, notes, examples).
3. URL becomes `#/word/<id>`.
4. Reload while at that hash — same card auto-opens.
5. Click a related link — referenced card opens inline and hash updates.
6. Toggle "Raw JSON" — object appears/hides.
7. Keyboard: Tab to card header and press Enter — card opens.
8. No uncaught errors in DevTools console.
9. Visual: confirm new Design System styling.

Acceptance
- All checklist steps pass on Chrome & Firefox mobile viewport.
- Any console errors are documented in `docs/issue_notes.md`.
